Cuestionario: Curso de Next.js 14 con Midudev

1. ¿Qué es Next.js y por qué se dice que es el framework de React más popular?
Next.js es un framework que usa React. Es el más popular porque está optimizado y tiene características listas para usar, como el renderizado en el servidor.

2. ¿Cuál es el comando exacto con npx para inicializar un proyecto de Next.js asegurando usar la última versión?
npx create-next-app@latest

3. ¿Cuál es el comando utilizado para levantar el entorno de desarrollo y en qué puerto se ejecuta por defecto?
npm run dev y se ejecuta en el puerto 3000 por defecto.

4. ¿Qué función cumple el comando npm run build en un proyecto de Next.js?
Genera una versión optimizada y lista para producción.

5. Describe para que sirven las carpetas, public y app
public es para los archivos visuales y estáticos como las imágenes. app es donde vive la aplicación, las rutas y donde se mueve React.

6. ¿Que es un CCS MODULE y como se usan?
Es un archivo CSS con alcance local. Se usa importándolo en el componente para aplicar estilos que no choquen ni sobrescriban a otros.

7. ¿Que hace el componente Layout.tsx ?
Envuelve a las páginas de forma global para mantener partes fijas sin que tengan que recargar al cambiar de página.

8. ¿Que es el enrutamiento?
Es la forma de navegar y mostrar distintas páginas mediante la URL.

9. ¿Cual es la diferencia entre Image e img?
Image es el componente de Next.js que optimiza las imágenes automáticamente. img es la etiqueta normal de HTML que carga la imagen sin optimizar.

10. ¿Que es el Lazy Load?
Es cargar recursos solo en el momento en que se necesitan o cuando aparecen en pantalla.

11. Cual es el funcionamiento del siguiente codigo, en que momento se muestra cada imagen
Hace que se muestre una imagen u otra dependiendo del tamaño de la pantalla (móvil o escritorio) para tener un diseño responsivo.

12. ¿Como funciona el enrutamiento en Next JS ?
Funciona creando carpetas dentro de app/, donde cada carpeta con su propio archivo page.tsx representa una ruta.

13. ¿Que es una ruta en el contexto web y donde las hemos visto en la clase ?
Es una página o vista a la que se accede mediante una URL. En la clase las vimos como carpetas anidadas dentro de app/.

14 y 15. ¿Que arrojo a MiduDev el localhost:3000/dashboard, donde hemos visto esos errores en la clase ?
Arrojó un error 404. Son errores de "página no encontrada" y suceden cuando se intenta entrar a una ruta que aún no hemos creado en el código.

16. ¿Que es el File System Routing?
Es hacer rutas por medio de archivos, o sea, la propia estructura de carpetas en tu proyecto representa las rutas de la web.

17. ¿Como funcionan las rutas anidadas?
Es cuando una ruta tiene dentro de sí otra ruta, es decir, creando una carpeta dentro de otra carpeta (ej. dashboard/customers).

18. Investiga por que es necesario esto (export default function)
Es necesario porque Next.js requiere una exportación por defecto para poder identificar exactamente qué componente debe renderizar como página o layout.

19. ¿Para que sirve el componente Link?
Hace que al navegar las rutas no tengan que cargar todos los elementos de la página desde cero, sino solo los que necesitan cambiar.

20. ¿Que se carga cuando usamos <a> y que cuando se usa <Link>?
Con <a> se recargan todos los elementos de la página. Con <Link> se hace una navegación interna optimizada y mucho más rápida.

21. Investiga que es un Hook
Es una función especial de React que te permite "engancharte" a sus funciones internas, como manejar estados.

22. ¿Que hooks usa Midu?
usePathname

23. ¿Por que ocurre el siguiente error? (Error: usePathname only works in Client Components...)
Es porque el componente está intentando usar algo del cliente (como la URL actual), pero por defecto está funcionando en el servidor. Se arregla agregando "use client".

24. ¿Es posible ejecutar los componentes de react en el cliente y el servidor?
Sí.

25. ¿Que es el code splitting?
Es la técnica que permite dividir el código en pequeñas partes para descargar solo lo que el usuario necesita en ese momento preciso.
